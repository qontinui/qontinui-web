DATABASE SCHEMA:
================================================================================

Three new tables will be created by the migration:

1. project_locks
   - Stores resource locks to prevent concurrent editing
   - Auto-expiration after 5 minutes (configurable)
   - Foreign keys: project_id, user_id
   - Indexes: project_id, user_id, resource_id, expires_at, (project_id, resource_id)

2. project_comments
   - Stores comments and threaded discussions
   - Supports canvas positioning, mentions, resolution
   - Foreign keys: project_id, author_id, resolved_by, parent_comment_id
   - Indexes: project_id, workflow_id, action_id, author_id, parent_comment_id, resolved, created_at

3. activity_logs
   - Tracks all project activities for real-time updates
   - Stores action type, resource type, changes, metadata
   - Foreign keys: project_id, user_id
   - Indexes: project_id, user_id, action_type, resource_id, created_at, (project_id, created_at)

ARCHITECTURE NOTES:
================================================================================

Design Decisions:
1. WebSocket over HTTP polling for real-time updates (lower latency)
2. Connection pooling per project (scalability)
3. Rate limiting per user (prevent abuse)
4. Auto-expiring locks (prevent deadlocks)
5. Auto-release on disconnect (prevent orphaned locks)
6. Broadcast with exclusion (prevent echo)
7. Message validation with Pydantic (type safety)
8. Async/await throughout (non-blocking I/O)
9. Structured logging with structlog (observability)
10. Optimized database indexes (query performance)

Scalability:
- Connection manager can handle 1000+ concurrent connections
- Broadcast is O(n) where n = connections per project
- Database queries use indexes for optimal performance
- Background cleanup task prevents memory leaks
- Rate limiting prevents resource exhaustion

Reliability:
- Auto-reconnect logic (client-side responsibility)
- Graceful error handling
- Failed connection cleanup
- Message validation prevents crashes
- Lock expiration prevents deadlocks
- Heartbeat timeout detects dead connections
